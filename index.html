vue + webpack 部署到 Nginx

发表于 2018-04-16 | 分类于 frontend | | 阅读次数

一、打包项目

router mode 我选择了 hash，因为后端没有对 history 配置，这里要注意的是 base 要填对

打包之后生成的 dist 文件夹我放在了 /www/dist/，最好不要放在各种中英文加符号的一大串目录中，避免不必要的 bug 吧

Nginx 配置

注意这行 try_files $uri $uri/ /index.html;，因为我们是单页面项目，URL 中请求的地址实际是不存在对应映射的，所以将这些请求都转发到 index 中，不做这一步的话会出现刷新 404 的问题

启动

输入 sudo nginx 就可以启动了，访问 localhost:8080/dist 就可以

Queue 接口

Java 队列实现了元素先进先出的规则，即第一个添加的元素第一个被移除。

Deque 接口

双向队列(Deque),是Queue的一个子接口，双向队列是指该队列两端的元素既能入队(offer)也能出队(poll),如果将Deque限制为只能从一端入队和出队，则可实现栈的数据结构。对于栈而言，有入栈(push)和出栈(pop)，遵循先进后出原则

Object peek() 检索第一个元素但不移除，当队列为空时返回null

ArrayDeque

ArrayDeque 提供了使用 deque 和 resizable-array 的功能，它继承自 AbstractCollection，实现了 Deque 接口

特点如下：

  • 能从两端添加或删除元素
  • 不能存在空元素
  • 在没有外部同步的情况下，不是线程安全的
  • 没有容量限制
  • 比 LinkedList 和 Stack 快

ArrayList

Array 是使用动态数组来存储元素的，它继承于 AbstractList，实现了 List 接口。

主要特点：

  • 可以有重复的元素
  • 元素是有序的
  • 非同步的
  • 可以随机访问
  • 插入删除效率低

类的声明

遍历集合的方法

  • Iterator
  • for-each

LinkedList

LinkedList 使用双向链接来存储数据，具有链表的结构，它继承自 AbstractList，实现 List 和 Deque 接口。

主要特点：

  • 可以有重复的元素
  • 元素是有序的
  • 非同步的
  • 插入删除效率高
  • 可以当做 list，stack 或 queue 使用

类的声明







ArrayList 和 LinkedList 的区别

两者都实现了 List 接口，都是有序的，并且都是非同步的。但也有如下的区别：

ArrayList LinkedList
使用动态数组存储元素 使用双向链表存储元素
操作效率低，因为当有一个元素插入或删除时，其他元素的位置都要改变 操作效率高，链表结构不需要改变元素位置
ArrayList 能当做 list 使用因为它只实现了 List 接口 LinkedList可以当做 List 或者 queue 使用是因为它实现了 List 和 Deque 接口
ArrayList 更适合存储和访问数据 LinkedList 更适合操作数据

HashSet

HashSet 是用来创建一个用哈希表存储的集合，它继承自 AbstractSet，实现了 Set 接口。

主要特点：

  • HashSet 通过哈希机制来存储数据
  • HashSet 中的元素不能重复

Set 和 List 的区别

List 中可以有重复的元素，而 Set 不可以。

类的声明




LinkedHashSet

LinkedHashSet 是一个实现了 Set 接口的哈希表（Hash table）和链接队列（Linked list），它继承自 HashSet，实现了 Set 接口。

主要特点：

  • 和 HashSet 一样不能有重复的元素
  • 提供 set 的所有操作，并且允许有空元素
  • 元素有序

类的声明










TreeSet

Tree 是一个使用树结构存储的集合，它继承自 AbstractSet，实现了 NavigableSet 接口。

主要特点：

  • 和 HashSet 一样不能有重复的元素
  • 访问和检索速度快
  • 元素有序
